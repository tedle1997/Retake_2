Goal

The Web Atelier Album is a photo sharing Web application, with the following features:

  1. upload, download and delete picture files
  2. take and upload pictures from mobile phones
  3. display picture thumbnails
  4. display full size picture
  5. picture slideshow: automatic
  6. picture slideshow: manual
  7. share link to picture
  8. edit picture title and description
  9. select or unselect favourite pictures
  10. display random picture
  11. apply CSS filters to pictures

The Web Atelier Album also supports the following extra-functional requirements:

  • Persistent storage: the content is stored persistently and survives both browser and server restarts
  • Single page application: the content is dynamically loaded and rendered directly in the browser

In this assignment, we will add support for this additional extra-functional requirement:

  • Real-time update: as the content is changed from any Web browser, it is not necessary to refresh the other Web browsers to see the effects of the changes

You will also add the following new feature:

  1. remote control for the picture slideshow

Tasks

To satisfy the requirements we suggest you develop your code following these steps while reusing and extending your previous assignment solutions as much as possible.

Task 1. Connect Socket

  1. When the browser opens the single-page application, it should open a socket connection to the server.
  2. Display on the web application page whether the socket is currently connected or disconnected (e.g., add a specific status element or change the page background color)

Task 2. Notify Updates

When one browser inserts a new image or updates the content of the database, the other connected browsers should be notified and automatically refresh the displayed information as follows.

  1. the gallery view should be refreshed
  2. the slideshow view should be refreshed (without restarting the slideshow)
  3. the editor view shoud be refreshed

Task 3. Notify Deletes

When one browser deletes one of the images, the other connected browsers should be notified and automatically refresh the currently displayed views.

  1. The thumbnail should disappear from the gallery view
  2. If the slideshow or editor views are currently displaying the deleted image, they should disappear

Task 4. Slideshow remote control

Select which image should be displayed in the slideshow from a different browser.

  1. In the gallery view add a Show button next to each image
  2. When the button is clicked, broadcast a message to all connected browsers.
  3. If the browsers are displaying the slideshow view, they should display the selected image

Task 5. Optional Features

  1. Synchronize the editor view as soon as the content of the form changes, without having to click on the submit button
  2. What if two editors are open at the same time on the same image in different browsers? Add a warning message stating that someone else is editing the same image
  3. Extend the behavior of the Delete button so that the image is deleted only after ALL users have clicked on the Delete button from all connected browsers. Count how many users have already clicked on the button and display the number (e.g., 0/N -> N/N)
  4. Extend the remote control so that users take turns in advancing the slideshow. It should not be possible for the same user to advance to the next image again before every one else did it
  5. Extend the remote control so that the slideshow will display the image selected by the majority of the connected users.
